# Database Guidelines

> Database patterns and conventions for this project.

---

## Overview

Two data stores:

| Store | Purpose | Library |
|-------|---------|---------|
| **PostgreSQL** | LangGraph checkpoints, session state | SQLAlchemy (async) + Alembic |
| **Qdrant** | Product embeddings, persona similarity search | `qdrant-client` |

PostgreSQL is the system of record. Qdrant is a specialized index -- treat it as a cache that can be rebuilt from the catalog seed data.

## Configuration

Database URLs and Qdrant settings are managed via `app/config.py`:

```python
# app/config.py:6-17
class Settings(BaseSettings):
    """Application settings loaded from environment variables."""

    database_url: str = "postgresql+asyncpg://postgres:postgres@localhost:5432/echo"
    qdrant_url: str = "http://localhost:6333"
    qdrant_collection: str = "products"
    ...

    model_config = {"env_file": ".env", "env_file_encoding": "utf-8"}
```

Environment variables override defaults. See `.env.example` for required variables.

## Query Patterns

### SQLAlchemy (PostgreSQL)

- Use async sessions via `AsyncSession` from `sqlalchemy.ext.asyncio`
- Inject sessions through FastAPI dependencies (`app/dependencies.py`)
- One session per request -- don't create sessions inside services

The session factory is implemented in `app/dependencies.py`:

```python
# app/dependencies.py:22-39
_engine_cache: dict[str, async_sessionmaker[AsyncSession]] = {}

def _get_session_factory(database_url: str) -> async_sessionmaker[AsyncSession]:
    """Create or return a cached async session factory."""
    if database_url not in _engine_cache:
        engine = create_async_engine(database_url, echo=False)
        _engine_cache[database_url] = async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)
    return _engine_cache[database_url]

async def get_db_session(
    settings: Annotated[Settings, Depends(get_settings)],
) -> AsyncGenerator[AsyncSession, None]:
    """Yield an async database session. One session per request."""
    factory = _get_session_factory(settings.database_url)
    async with factory() as session:
        yield session
```

Key details:
- Engine is cached per `database_url` in `_engine_cache` dict -- avoids recreating engines on every request
- `expire_on_commit=False` prevents attribute access errors after commit
- `get_db_session` is an async generator that yields and auto-closes via `async with`
- Session is injected via `Annotated[AsyncSession, Depends(get_db_session)]` in router parameters

```python
# Good: session injected via Depends
async def get_checkpoint(session: AsyncSession, session_id: str) -> Checkpoint | None:
    result = await session.execute(select(Checkpoint).where(Checkpoint.session_id == session_id))
    return result.scalar_one_or_none()

# Bad: session created inside function
async def get_checkpoint(session_id: str):
    async with async_session() as session:  # Don't do this
        ...
```

### Qdrant

- Use `qdrant_client.AsyncQdrantClient` (see `app/dependencies.py:42-46`)
- Collection name: `products` (configured in `Settings.qdrant_collection`)
- Always pass `limit` and `score_threshold` to search calls

Qdrant client is injected via FastAPI dependency:

```python
# app/dependencies.py:42-46
async def get_qdrant_client(
    settings: Annotated[Settings, Depends(get_settings)],
) -> AsyncQdrantClient:
    """Return an async Qdrant client."""
    return AsyncQdrantClient(url=settings.qdrant_url)
```

Service functions accept the client as a parameter:

```python
# app/services/recommendation.py:11-18
async def get_recommendations(
    persona_embedding: list[float],
    qdrant_client: AsyncQdrantClient,
    collection: str,
    limit: int = 10,
    score_threshold: float = 0.5,
) -> list[Recommendation]:
    """Retrieve product recommendations from Qdrant based on persona embedding."""
```

```python
# app/services/catalog.py:9-13
async def seed_catalog(
    qdrant_client: AsyncQdrantClient,
    collection: str,
) -> int:
    """Seed the product catalog into Qdrant. Returns number of products seeded."""
```

## Migrations

- Tool: **Alembic** with async support
- Location: `backend/alembic/versions/`
- Config: `backend/alembic.ini` (sqlalchemy.url left blank -- overridden by `Settings` in `alembic/env.py`)
- Create: `alembic revision --autogenerate -m "add checkpoints table"`
- Apply: `alembic upgrade head`
- Always review autogenerated migrations before committing

### Async Alembic Setup (`alembic/env.py`)

The Alembic env uses async engine via `async_engine_from_config` and `connection.run_sync`:

```python
# alembic/env.py:24-26
# Override sqlalchemy.url from application settings
settings = Settings()
config.set_main_option("sqlalchemy.url", settings.database_url)
```

```python
# alembic/env.py:55-66
async def run_async_migrations() -> None:
    """Run migrations in 'online' mode with an async engine."""
    connectable = async_engine_from_config(
        config.get_section(config.config_ini_section, {}),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )

    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)

    await connectable.dispose()
```

When SQLAlchemy models are defined, uncomment the `Base` import in `alembic/env.py:21` and set `target_metadata = Base.metadata`.

## Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Tables | `snake_case`, plural | `checkpoints`, `sessions` |
| Columns | `snake_case` | `session_id`, `created_at` |
| Indexes | `ix_{table}_{column}` | `ix_checkpoints_session_id` |
| Foreign keys | `fk_{table}_{ref_table}` | `fk_checkpoints_sessions` |
| Primary keys | `id` (UUID) | |
| Timestamps | `created_at`, `updated_at` | Always UTC, timezone-aware |

## Common Mistakes

- **Don't store embeddings in PostgreSQL** -- they go in Qdrant. PostgreSQL stores metadata only.
- **Don't skip migrations** -- even for "small" schema changes. Alembic tracks everything.
- **Don't use raw SQL** unless there's a measurable performance reason. Use SQLAlchemy's query builder.
- **Don't forget `await`** on async session operations -- this produces silent bugs.
- **Don't create sessions inside service functions** -- inject via `get_db_session` dependency.
- **Don't hardcode database URLs** -- always use `Settings` from `app/config.py`.
