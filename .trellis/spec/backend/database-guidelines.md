# Database Guidelines

> Database patterns and conventions for this project.

---

## Overview

Two data stores:

| Store | Purpose | Library |
|-------|---------|---------|
| **PostgreSQL** | LangGraph checkpoints, session state | SQLAlchemy (async) + Alembic |
| **Qdrant** | Product embeddings, persona similarity search | `qdrant-client` |

PostgreSQL is the system of record. Qdrant is a specialized index — treat it as a cache that can be rebuilt from the catalog seed data.

---

## Query Patterns

### SQLAlchemy (PostgreSQL)

- Use async sessions via `AsyncSession` from `sqlalchemy.ext.asyncio`
- Inject sessions through FastAPI dependencies (`dependencies.py`)
- One session per request — don't create sessions inside services

```python
# Good: session injected
async def get_checkpoint(session: AsyncSession, session_id: str) -> Checkpoint | None:
    result = await session.execute(select(Checkpoint).where(Checkpoint.session_id == session_id))
    return result.scalar_one_or_none()

# Bad: session created inside function
async def get_checkpoint(session_id: str):
    async with async_session() as session:  # Don't do this
        ...
```

### Qdrant

- Use `qdrant_client.AsyncQdrantClient`
- Collection name: `products`
- Vector name: `clip_embedding`
- Always pass `limit` and `score_threshold` to search calls

## Migrations

- Tool: **Alembic** with async support
- Location: `backend/alembic/versions/`
- Naming: auto-generated with descriptive message
- Create: `alembic revision --autogenerate -m "add checkpoints table"`
- Apply: `alembic upgrade head`
- Always review autogenerated migrations before committing

## Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Tables | `snake_case`, plural | `checkpoints`, `sessions` |
| Columns | `snake_case` | `session_id`, `created_at` |
| Indexes | `ix_{table}_{column}` | `ix_checkpoints_session_id` |
| Foreign keys | `fk_{table}_{ref_table}` | `fk_checkpoints_sessions` |
| Primary keys | `id` (UUID) | |
| Timestamps | `created_at`, `updated_at` | Always UTC, timezone-aware |

## Common Mistakes

- **Don't store embeddings in PostgreSQL** — they go in Qdrant. PostgreSQL stores metadata only.
- **Don't skip migrations** — even for "small" schema changes. Alembic tracks everything.
- **Don't use raw SQL** unless there's a measurable performance reason. Use SQLAlchemy's query builder.
- **Don't forget `await`** on async session operations — this produces silent bugs.
